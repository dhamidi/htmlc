# Enhance Blog Editor Form Fields for a Better Writing Experience

## Description

The post editor form has basic neo-brutalist field styling but lacks the polish expected for a writing tool. The title input has no placeholder, the textarea is too short for editing real post bodies, and there is no visual distinction between the title and body fields. This task improves `PostForm.vue` and the form pages to make the editor feel intentional and comfortable for writers.

Changes are confined to `examples/blog/templates/`:

- **`PostForm.vue`**:
  - Add `placeholder="Post title"` to the title `<input>`.
  - Add `placeholder="Write your post here…"` to the `<textarea>`.
  - Increase `textarea` `min-height` to `16rem` (was 8rem).
  - Style the title input larger than the body textarea to reflect its role: `font-size: 1.25rem`, `font-weight: 700`.
  - Add a `box-shadow: inset 2px 2px 0 #eee` to resting inputs to give them a subtle depth cue (different from the outer focus shadow).
  - Tighten the focus style: on focus, remove the inset shadow and apply `box-shadow: 5px 5px 0 #000` with `outline: none`.
  - Add `transition: box-shadow 0.1s` for a smooth focus animation.

- **`PostFormPage.vue`** (new/edit pages): Wrap the form in a `<div class="editor-shell">` with `max-width: 720px; margin: 0 auto` so the editor does not stretch across wide viewports.

## Acceptance Criteria

- Title `<input>` has `placeholder="Post title"` attribute in rendered HTML
- Body `<textarea>` has `placeholder="Write your post here…"` attribute in rendered HTML
- `textarea` `min-height` is at least `16rem` in the CSS
- Title `<input>` has a larger `font-size` than the default body text (≥ 1.2rem)
- `PostForm.vue` CSS includes a `transition` property on inputs/textarea
- `PostFormPage.vue` or equivalent wraps the form in a container with `max-width` ≤ 760px
- All existing server tests (`go test ./examples/blog/...`) continue to pass after the changes
