# Convert Engine rendering to io.Writer

## Description

With the Renderer now writing to `io.Writer` (task 005), update the Engine's public rendering methods to also accept an `io.Writer` as their primary interface, and provide `*String` convenience methods for callers that need strings.

### Changes to `engine.go`

1. Change `renderComponent` (internal) to accept `io.Writer` and pass it through to `Renderer.Render`. It still needs to return the `*StyleCollector` so that `RenderPage` and `RenderFragment` can inject styles.

   For `RenderFragment`, styles can be written to `w` before the component output, so the component can render directly to `w` after the style block is written.

   For `RenderPage`, the output must be buffered internally because the `</head>` insertion point is discovered by scanning the rendered HTML. Buffer the component output into a `strings.Builder`, locate `</head>`, and write the combined result (styles + HTML) to `w`.

2. Change `RenderPage` signature from `RenderPage(name string, data map[string]any) (string, error)` to `RenderPage(w io.Writer, name string, data map[string]any) error`.

3. Change `RenderFragment` signature from `RenderFragment(name string, data map[string]any) (string, error)` to `RenderFragment(w io.Writer, name string, data map[string]any) error`.

4. Add `RenderPageString(name string, data map[string]any) (string, error)` convenience method.

5. Add `RenderFragmentString(name string, data map[string]any) (string, error)` convenience method.

6. Update `ServeComponent` to call `RenderFragment(w, name, scope)` writing directly to the `http.ResponseWriter` instead of building a string and calling `fmt.Fprint`.

7. Update `styleBlock` helper to accept `io.Writer` and write directly, or keep it returning a string if the caller (RenderPage) needs to do string manipulation for `</head>` insertion.

8. Update all tests in `engine_test.go` and `integration_test.go` to use the new signatures (`RenderPageString` / `RenderFragmentString` for tests that compare output strings).

## Acceptance Criteria

- `Engine.RenderPage(w, name, data)` writes the full page HTML (with styles injected before `</head>`) directly to `w` and returns only an error.
- `Engine.RenderFragment(w, name, data)` writes the fragment HTML (with styles prepended) directly to `w` and returns only an error.
- `Engine.RenderPageString(name, data)` and `Engine.RenderFragmentString(name, data)` return `(string, error)` with identical output to the old methods.
- `ServeComponent` writes directly to `http.ResponseWriter` without intermediate string allocation.
- All existing tests in `engine_test.go` and `integration_test.go` pass after being updated.
- `go vet ./...` and `go build ./...` pass with no errors.
