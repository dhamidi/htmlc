# Add comprehensive tests and examples for slots

## Description

Add thorough test coverage and documentation examples for the complete named and scoped slots feature. This task ensures all edge cases are covered, integration scenarios work correctly, and the public API is well-documented through example tests.

**Tests to add to `renderer_test.go`** (if not already covered by tasks 001-004):

Binding pattern edge cases:
- Binding pattern with extra whitespace everywhere: `"  {  a  ,  b  }  "`
- Empty binding (just `v-slot` with no `=`) works

Named slot edge cases:
- Multiple named slots with same name — last one wins (or first one wins, match Vue behavior)
- `<template>` without any directive renders its children transparently (not treated as a slot)
- `<template #default>` explicitly naming the default slot works

Scoped slot edge cases:
- Scoped slot where slot props map has keys not in the binding pattern — extra keys ignored
- Scoped slot where binding pattern has keys not in slot props — those variables are `nil`
- Deeply nested: component A uses component B with named slots, component B uses component C with scoped slots

**Tests to add to `integration_test.go`:**

- Layout component pattern: `<Layout>` with header/main/footer named slots, used by a page component
- Renderless list pattern: `<UserList :users="users">` with `<template #item="{ user, index }">` inside `v-for`
- Nested components with slots: parent → child → grandchild, each level using slots

**Tests to add to `example_test.go`:**

- `ExampleRender_namedSlots` — demonstrates `<template #header>` / `<template #footer>` with a layout component
- `ExampleRender_scopedSlots` — demonstrates child passing data to parent via `:attr` on `<slot>` and `v-slot="{ ... }"` on caller
- `ExampleRender_slotFallbackContent` — demonstrates fallback content when no slot content is provided
- `ExampleRender_singleVariableSlotBinding` — demonstrates `v-slot="slotProps"` binding the entire props object

## Acceptance Criteria

- All edge cases listed above have passing tests
- Integration tests verify multi-component slot scenarios work end-to-end
- Example tests in `example_test.go` serve as documentation and match the examples from the specification (Named Slots, Scoped Slots, Fallback Content, Single Variable Binding)
- Example test output comments match actual rendered output
- `go test ./...` passes with no failures
- No test uses `$slot` directly — all slot behavior goes through the new `SlotDefinition` mechanism
