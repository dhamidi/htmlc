# Add Comprehensive Tests for Named and Scoped Slots

## Description

Write full test coverage for the slot system across three test files. Tests must cover every behavioural guarantee described in the specification, including happy paths, edge cases, and error conditions. All tests must pass with the implementation from Tasks 001–005.

### renderer_test.go — unit tests

Add the following test cases (each as a separate `Test*` function or table-driven sub-test):

**Named slots:**
- `TestRender_NamedSlots` — component with `<slot name="header" />` and `<slot name="footer" />` receives content via `<template #header>` and `<template #footer>`; unnamed content goes to `<slot />` (default).
- `TestRender_NamedSlotsHashSyntax` — `#name` shorthand is equivalent to `v-slot:name`.
- `TestRender_SlotFallbackWhenMissing` — a slot with fallback children renders those children when no caller content targets that slot name.
- `TestRender_NamedSlotOverridesFallback` — when the caller does provide content for a named slot, the fallback is not rendered.
- `TestRender_NamedSlotExplicitDefault` — `<template #default>` explicitly targets the default slot.
- `TestRender_MultipleNamedSlotsSameName_LastWins` — duplicate `<template #header>` blocks: last definition wins.
- `TestRender_TemplatePlainTransparent` — a `<template>` with no slot directive inside a component renders its children without a wrapper element.

**Scoped slots:**
- `TestRender_ScopedSlotDestructured` — `v-slot="{ user, index }"`: destructured variables are available in slot content.
- `TestRender_ScopedSlotSingleVar` — `v-slot="slotProps"`: full slot props object bound to one variable.
- `TestRender_ScopedSlotNoBinding` — `v-slot` with no value (no `=`): slot renders without error even though child passes props.
- `TestRender_ScopedSlotStaticAttr` — static (non-dynamic) attribute on `<slot static-key="value" />` is available as a slot prop.
- `TestRender_ScopedSlotInsideVFor` — scoped slot inside `v-for` in the child; each iteration provides independent slot props.
- `TestRender_ScopedSlotParentScopeAccessible` — slot content can reference both parent scope variables and slot props simultaneously.
- `TestRender_ScopedSlotPropOverridesParentVar` — when a slot prop name collides with a parent scope variable, the slot prop takes precedence within the slot content.
- `TestRender_ScopedSlot_ExtraKeysIgnored` — destructure pattern with fewer keys than the child passes; extra keys are silently ignored.
- `TestRender_NamedScopedSlot` — `<template #item="{ user }">` on a named slot with scoped data.

**`v-slot` on component tag:**
- `TestRender_VSlotOnComponentTag_Destructured` — `<MyComp v-slot="{ item }">` wraps all children as default slot with destructured binding.
- `TestRender_VSlotOnComponentTag_SingleVar` — `<MyComp v-slot="props">` wraps all children as default slot with single-variable binding.
- `TestRender_VSlotOnComponentTag_HashSyntax` — `<MyComp #default="{ item }">` is equivalent.
- `TestRender_VSlotOnComponentTag_MixedError` — combining `v-slot` on the tag with a `<template #namedSlot>` child produces a render error.

**Binding pattern:**
- `TestParseBindingPattern` — table-driven: empty, single identifier, `{ a }`, `{ a, b, c }`, whitespace variants, invalid patterns all produce correct results.

**Deeply nested and backward compatibility:**
- `TestRender_DeeplyNestedSlots` — three levels of component nesting with slots at each level.
- `TestRender_ComponentSlot` (existing) — must still pass unchanged.
- `TestRender_ComponentSlotWithExpression` (existing) — must still pass unchanged.

### integration_test.go — end-to-end tests

- `TestIntegration_LayoutPattern` — full layout component with header, body, and footer named slots; page component fills all three; correct HTML output verified.
- `TestIntegration_RenderlessListPattern` — `UserList` component iterates over `users` with `v-for`, passes `user` and `index` through a scoped slot; caller renders a custom `<li>` template; full HTML output verified.
- `TestIntegration_NestedComponentsWithSlots` — parent passes content through an intermediate component to an inner component's slot; slot props propagate correctly through all levels.
- `TestIntegration_CamelCasePropViaSlot` — component prop passed via slot binding works correctly even with HTML parser attribute lowercasing.

### example_test.go — documentation examples

- `ExampleRenderer_namedSlots` — mirrors the Named Slots example from the specification; `// Output:` comment included so `go test` verifies the output.
- `ExampleRenderer_scopedSlots` — mirrors the Scoped Slots example from the specification with the `Output:` comment.

## Acceptance Criteria

- `go test ./...` passes with zero failures.
- Every test function listed above exists and exercises the described scenario.
- `ExampleRenderer_*` functions include `// Output:` blocks whose expected output matches the specification's examples exactly (modulo whitespace normalisation already in use by the test suite).
- No existing tests are deleted or modified to work around a regression.
- Test helpers / fixtures are added inline or as unexported helpers within the test file; no new non-test source files are created.
