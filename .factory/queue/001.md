# Remove `len` builtin and expose `RegisterBuiltin` API in expr package

## Description

The `len()` function is currently a built-in in the expr evaluator (`expr/eval.go`). Vue.js 3 does not have a global `len` function — arrays expose their size via `.length`, which the evaluator already supports via `accessMember`. Remove `len` from the `builtins` map so the expression language matches Vue.js 3 semantics.

At the same time, keep the `builtins` infrastructure intact and make it accessible to callers: export a `RegisterBuiltin(name string, fn func(...any) (any, error))` function so that host applications can register custom functions (e.g. `translate`, `formatDate`) that are available in all template expressions without being passed in the scope on every render call.

Files to change: `expr/eval.go`

## Acceptance Criteria

- `expr.Eval("len(items)", scope)` returns an error (len is not callable — it resolves to `undefined` which is not a function)
- `expr.Eval("items.length", scope)` continues to return the correct length for slice/array values
- `expr.RegisterBuiltin("double", func(args ...any)(any,error){ ... })` makes `double(x)` callable in subsequent `Eval` calls
- The `builtins` variable remains unexported; only `RegisterBuiltin` is exported
- `go test ./expr/...` passes (existing tests not yet updated are expected to fail — those are handled in the next task)
