# Blog example: data model and in-memory store

## Description

Create the `examples/blog` subpackage with a `Post` struct and an in-memory store that supports creating, reading, updating, and listing blog posts. This is the foundation the HTTP handlers and Vue components depend on.

The package should live at `examples/blog/` inside the repository. The store must be safe for use from a single goroutine (no concurrency required). Posts are identified by a sequential integer ID assigned at creation time.

The `Post` struct must have at minimum:
- `ID int`
- `Title string`
- `Body string`
- `CreatedAt time.Time`

The store must expose:
- `NewStore() *Store` — constructor returning an empty store
- `(*Store) Create(title, body string) *Post` — assigns next ID, sets CreatedAt to now, appends to store, returns pointer to new post
- `(*Store) List() []*Post` — returns all posts in insertion order
- `(*Store) Get(id int) (*Post, bool)` — returns a post by ID and an ok flag
- `(*Store) Update(id int, title, body string) (*Post, bool)` — updates title/body of an existing post, returns updated post and ok flag; returns false if not found

## Acceptance Criteria

- File `examples/blog/store.go` exists and belongs to package `blog`
- `Post` struct has fields `ID int`, `Title string`, `Body string`, `CreatedAt time.Time`
- `NewStore()` returns a non-nil `*Store` with zero posts
- `Create` assigns sequential IDs starting from 1
- `List` returns posts in creation order
- `Get` returns `(nil, false)` for unknown IDs
- `Update` returns `(nil, false)` for unknown IDs and updates the post in-place for known IDs
- The file compiles with `go build ./examples/blog/`
