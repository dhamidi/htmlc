# Add neo-brutalist base CSS to Layout.vue

## Description

Layout.vue is the outermost page shell for the blog. It wraps all pages and provides the `header`, default, and `footer` slot positions. Currently the component has no styles — the rendered HTML is completely unstyled.

Add a `<style>` section to Layout.vue containing the global CSS foundation for the neo-brutalist design system: a CSS reset, base typography, black-and-white color palette, and structural styles for the `<header>`, `<main>`, and `<footer>` elements. Also update the template to add CSS classes to the structural elements so the styles have targets to attach to.

The engine's `RenderPage` function automatically injects collected `<style>` blocks before `</head>`, so styles written in any component's `<style>` section are available in the final document without any server changes.

Neo-brutalist design principles to apply:
- Background: `#fff`, text: `#000`
- `font-family`: a bold system sans-serif stack (`"Arial Black", Arial, sans-serif`) or similar high-impact choice
- No border-radius anywhere
- `<header>`: thick bottom border (`3px solid #000`), bold site name, good padding, uppercase tracking
- `<main>`: centered max-width container, generous vertical padding
- `<footer>`: thick top border (`3px solid #000`), small text, full-width

## Acceptance Criteria

- Layout.vue has a `<style>` section with a CSS reset (`*, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }`) and body base styles
- `<body>` has `background: #fff`, `color: #000`, and a bold sans-serif font family
- The `<header>` element has a thick solid black bottom border, meaningful padding, and the slot content is visually prominent
- The `<main>` element uses a centered max-width layout (e.g. `max-width: 800px; margin: 0 auto`)
- The `<footer>` element has a thick solid black top border and padding
- CSS classes are added to the relevant elements in the template (e.g. `class="site-header"`, `class="site-main"`, `class="site-footer"`)
- `go test ./examples/blog/...` passes after the change
