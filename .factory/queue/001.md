# Add SlotDefinition type and binding pattern parser

## Description

Add the foundational types and parsing logic needed for named and scoped slots. This task introduces the `SlotDefinition` struct and a `parseBindingPattern` function that interprets the value of `v-slot` / `#` directives into either no binding, a single variable binding, or a destructured binding.

The `SlotDefinition` type captures everything needed to defer slot rendering: the AST nodes from the caller's template, the caller's scope at invocation time, and the parsed binding information. This type will be used by subsequent tasks to replace the pre-rendered `$slot` string mechanism.

The `parseBindingPattern` function handles these cases:
- Empty/absent value → no bindings (`BindingVar` and `Bindings` both empty)
- Single identifier (e.g. `props`) → `BindingVar = "props"`
- Destructured (e.g. `{ user, index }`) → `Bindings = ["user", "index"]`
- Anything else → returns an error

Add both to `renderer.go`.

Also add a helper function `parseSlotDirective` that takes an attribute name string and returns `(slotName string, isSlotDirective bool)`. It should recognize:
- `v-slot` → name `"default"`, true
- `v-slot:header` → name `"header"`, true
- `#header` → name `"header"`, true
- `#default` → name `"default"`, true
- Anything else → `"", false`

## Acceptance Criteria

- `SlotDefinition` struct is defined in `renderer.go` with fields: `Nodes []*html.Node`, `ParentScope map[string]any`, `BindingVar string`, `Bindings []string`
- `parseBindingPattern(s string) (bindingVar string, bindings []string, err error)` is implemented and handles: empty string, single identifier, destructured `{ a, b }`, and invalid patterns
- `parseSlotDirective(attrName string) (slotName string, isSlotDirective bool)` is implemented
- Tests in `renderer_test.go` cover:
  - Empty pattern returns no binding var and no bindings
  - Single identifier pattern (e.g. `"props"`) returns `bindingVar="props"`
  - Destructured pattern `"{ user, index }"` returns `bindings=["user", "index"]`
  - Whitespace is tolerated: `"{  user ,  index  }"` works
  - Single-variable destructure `"{ item }"` works
  - Invalid patterns (e.g. `"{ }"`, `"{ a, }"`, `"123"`, `"{ a b }"`) return errors
  - `parseSlotDirective` correctly parses `v-slot`, `v-slot:name`, `#name`, `#default`, and rejects non-slot attributes
- All existing tests continue to pass
