# Update documentation to reflect current engine state

## Description

Several documentation files still describe features that no longer exist or
features that have since been implemented. This task brings all docs up to date:

1. **Remove stale `len` builtin references.** The `len` function was removed
   from the expression evaluator's built-in table. Every reference to it as a
   built-in must be replaced with accurate information. The engine now exposes
   `expr.RegisterBuiltin` for callers who want to add custom functions, and
   the `.length` property already works on strings, slices, arrays, and maps
   via member-access syntax. Locations to fix:
   - `README.md` — expression-language table row (`Function calls | len(items)`),
     the Built-in functions table (`len(x)` row), and the examples block
     (`{{ len(tags) }}`).
   - `expr/doc.go` — the Identifiers section says "currently only `len`"; the
     entire `## len(value) float64` built-in section should be replaced with a
     description of `RegisterBuiltin` and a note that `.length` handles the
     common case.

2. **Document `v-slot` as supported.** Named and scoped slots are now fully
   implemented but the docs still list them as "Not supported":
   - `README.md` directives table: the `v-slot` row currently reads
     "Not supported. Only the default `<slot />` is available." Replace with a
     brief description and a cross-reference to the new component-system section.
   - `README.md` component-system "Not supported" table: remove the
     "Named slots / scoped slots / `v-slot`" row.
   - Add a new **Slots** subsection under §4 Component System that documents:
     - Default slot (already partially covered — keep and expand)
     - Named slots (`<slot name="…" />` + `<template #name>`)
     - Scoped slots (`:prop="expr"` on `<slot>`, `v-slot="{ … }"` on caller)
     - Fallback content (children of `<slot>` rendered when caller provides none)
     - The `#shorthand` (`#name` = `v-slot:name`)
     - Binding patterns: empty, single variable, destructured `{ a, b }`
     - Scope rules: slot content evaluated in caller's scope; slot props merged in

## Acceptance Criteria

- `README.md` contains no references to `len` as a built-in function.
- `expr/doc.go` Identifiers section no longer states "currently only `len`";
  the `## len(value) float64` section is removed and replaced with a
  `RegisterBuiltin` description and a note about `.length`.
- `README.md` directive table `v-slot` row describes the directive as supported.
- `README.md` component-system "Not supported" table no longer lists named/scoped
  slots.
- `README.md` contains a Slots subsection under §4 with working examples for
  default slot, named slots, scoped slots, fallback content, and the `#` shorthand.
- All existing tests continue to pass (`go test ./...`).
