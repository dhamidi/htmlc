# Add CollectIdentifiers to expr package

## Description

Add a new exported function `CollectIdentifiers(src string) ([]string, error)` to the `expr` package in a new file `expr/identifiers.go`. This function compiles the given expression source and walks the resulting AST to collect all `Identifier` node names, applying the following rules:

- `Identifier` → collect the name.
- `MemberExpr` with `Computed=false` (dot notation) → walk `Object` only, skip `Property`.
- `MemberExpr` with `Computed=true` (bracket notation) → walk both `Object` and `Property`.
- `UnaryExpr` → walk `Operand`.
- `BinaryExpr` → walk `Left` and `Right`.
- `TernaryExpr` → walk `Condition`, `Consequent`, and `Alternate`.
- `CallExpr` → walk `Callee` and all `Args`.
- `ArrayLit` → walk all `Elements`.
- `ObjectLit` → walk all property `Value` nodes (keys are string literals, not identifiers).
- Literal nodes (`NumberLit`, `StringLit`, `BoolLit`, `NullLit`, `UndefinedLit`) → skip.

Duplicates are removed. Order is unspecified.

Add tests in `expr/identifiers_test.go` covering:

- Simple identifier: `"x"` → `["x"]`
- Dot-notation member access: `"user.name"` → `["user"]`
- Bracket-notation member access: `"items[idx]"` → `["items", "idx"]`
- Binary expression: `"a + b"` → `["a", "b"]`
- Ternary expression: `"x ? y : z"` → `["x", "y", "z"]`
- Unary expression: `"!flag"` → `["flag"]`
- Call expression: `"len(items)"` → `["len", "items"]`
- Array literal: `"[a, b, 1]"` → `["a", "b"]`
- Object literal: `"{key: val}"` → `["val"]` (key is a string literal, not an identifier)
- Nested member + bracket: `"a.b[c].d"` → `["a", "c"]`
- Duplicate removal: `"x + x"` → `["x"]`
- Compile error returns error

## Acceptance Criteria

- `expr.CollectIdentifiers` is exported and returns deduplicated identifier names from the expression AST.
- Dot-notation property names (e.g. `name` in `user.name`) are excluded; bracket-notation property expressions are included.
- All tests in `expr/identifiers_test.go` pass.
