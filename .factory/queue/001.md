# Define SlotDefinition Type and Parsing Utilities

## Description

Introduce the foundational data structures and parsing helpers that all subsequent slot work depends on. This task adds three things to `renderer.go`:

1. **`SlotDefinition` struct** — captures the deferred slot content supplied by a caller: the raw AST nodes, the caller's scope at invocation time, and the parsed binding pattern that describes how slot props should be made available to the slot content.

2. **`parseBindingPattern(value string) (bindingVar, bindings, error)`** — turns the string value of a `v-slot` or `#name` attribute into one of three binding forms:
   - *Empty* → no bindings (slot props are discarded).
   - *Single identifier* (e.g. `slotProps`) → whole slot-props object bound to one variable.
   - *Destructure list* (e.g. `{ user, index }`) → individual variables extracted from the slot-props object.
   - Anything else → render error.

3. **`parseSlotDirective(attrName string) (slotName string, ok bool)`** — recognises the four syntactic forms a slot directive can take on a `<template>` or component tag and returns the normalised slot name:
   - `v-slot` → `"default"`
   - `v-slot:header` → `"header"`
   - `#default` → `"default"`
   - `#footer` → `"footer"`
   - Any other attribute → `ok = false`

These helpers are pure functions with no side-effects, making them easy to unit-test in isolation and straightforward to call from the more complex rendering logic introduced in later tasks.

## Acceptance Criteria

- `SlotDefinition` is declared in `renderer.go` with fields `Nodes []*html.Node`, `ParentScope map[string]any`, `BindingVar string`, and `Bindings []string`.
- `parseBindingPattern("")` returns `("", nil, nil)` (no binding, no error).
- `parseBindingPattern("props")` returns `("props", nil, nil)`.
- `parseBindingPattern("{ user, index }")` returns `("", ["user", "index"], nil)`.
- `parseBindingPattern("{ a , b , c }")` handles arbitrary whitespace around identifiers without error.
- `parseBindingPattern("invalid!!pattern")` returns a non-nil error.
- `parseSlotDirective("v-slot")` returns `("default", true)`.
- `parseSlotDirective("v-slot:header")` returns `("header", true)`.
- `parseSlotDirective("#footer")` returns `("footer", true)`.
- `parseSlotDirective("#default")` returns `("default", true)`.
- `parseSlotDirective("class")` returns `("", false)`.
- All of the above are covered by unit tests.
