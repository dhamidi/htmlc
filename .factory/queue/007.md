# Add comprehensive Example tests showcasing all features

## Description

The existing `example_test.go` contains a single `Example()` function that demonstrates basic Engine usage with `RenderFragment`. Add a comprehensive set of `Example*` functions in `example_test.go` that showcase every major feature of the htmlc component system. These examples serve as both executable documentation (verified by `go test`) and as the primary learning resource in godoc output.

Each example should be a self-contained `Example*` function in `example_test.go` (package `htmlc_test`) that uses `// Output:` comments so `go test` verifies the output. Examples should use inline template strings via `ParseFile` or temporary directories with `Engine` as appropriate. Avoid depending on external fixture files.

### Examples to add

1. **ExampleParseFile** — Parse a `.vue` string into a `*Component` and print its `Path` to confirm parsing succeeded.

2. **ExampleRender_interpolation** — `{{ expr }}` text interpolation with member access (e.g. `{{ user.name }}`), arithmetic (`{{ price * qty }}`), and ternary (`{{ active ? "yes" : "no" }}`).

3. **ExampleRender_vText** — `v-text="expr"` replaces element children with HTML-escaped text.

4. **ExampleRender_vHtml** — `v-html="expr"` renders raw HTML without escaping.

5. **ExampleRender_vIf** — `v-if` / `v-else-if` / `v-else` conditional chain showing only the first truthy branch renders.

6. **ExampleRender_vFor** — `v-for="item in items"` iterating an array, `(item, index) in items` with index, and `n in 5` integer range.

7. **ExampleRender_vForObject** — `v-for="(value, key) in obj"` iterating map entries.

8. **ExampleRender_vBind** — `:href="url"` dynamic attribute binding, `:disabled="false"` boolean attr omission, and `:disabled="true"` boolean attr presence.

9. **ExampleRender_vBindClass** — `:class="{ active: isActive, hidden: isHidden }"` object syntax and `:class="['base', condition ? 'extra' : '']"` array syntax, including merge with static `class="..."`.

10. **ExampleRender_vBindStyle** — `:style="{ color: 'red', fontSize: '14px' }"` object syntax with camelCase-to-kebab conversion.

11. **ExampleRender_vShow** — `v-show="false"` injects `display:none`; `v-show="true"` renders normally.

12. **ExampleRender_vPre** — `v-pre` emits mustache syntax literally without interpolation.

13. **ExampleRender_componentSlot** — A parent component passes inner HTML content to a child component's `<slot />`.

14. **ExampleRender_componentProps** — A parent passes `:title="expr"` and static `class="x"` to a child component.

15. **ExampleComponent_Props** — Call `component.Props()` and print the discovered prop names and their expressions.

16. **ExampleRenderer_WithMissingPropHandler** — Render with `SubstituteMissingProp` to show placeholder output for missing props.

17. **ExampleEngine_RenderPage** — Full-page rendering with `<html><head></head><body>` template showing style injection before `</head>`.

18. **ExampleEngine_ServeComponent** — Create an `http.Handler` via `ServeComponent` and demonstrate an HTTP roundtrip using `httptest`.

19. **ExampleRender_eventPassthrough** — `@click="handler"` and `v-model="name"` are preserved in output as-is (client-side directives).

20. **ExampleEngine_scopedStyles** — A component with `<style scoped>` produces scoped CSS selectors and scope attributes on HTML elements.

## Acceptance Criteria

- All examples are `Example*` functions in `example_test.go` using package `htmlc_test`.
- Every example has a `// Output:` comment that `go test` verifies.
- `go test ./...` passes with all examples producing expected output.
- Examples collectively cover: interpolation, v-text, v-html, v-if/v-else-if/v-else, v-for (array, object, integer), v-bind (regular, class object/array, style object, boolean attrs), v-show, v-pre, component composition with slots, component props, `Component.Props()`, missing prop handling, Engine RenderPage with style injection, ServeComponent HTTP handler, client-side directive passthrough (@click, v-model), and scoped styles.
- No example depends on external fixture files; all use inline template strings or `t.TempDir()` / `os.MkdirTemp`.
- Examples are ordered from simple to complex to serve as a progressive tutorial.
