# Improve Blog Reading Typography and Page Layout

## Description

The blog currently uses "Arial Black" as the primary font across all text, which is excellent for headings and labels but poor for reading body copy. The layout also lacks a title tag and a navigation link back to the post list from form pages. This task improves the reading experience by introducing a readable body font, better line heights, and a minimal site navigation in the header.

Changes are confined to `examples/blog/templates/`:

- **`Layout.vue`**: Add `<title>` to `<head>`. Change `body` font to a readable serif or system-ui stack. Keep "Arial Black" only for `.site-header` and headings. Add an `<a href="/">` home link inside `.site-header`. Increase body line height to at least 1.6.
- **`PostCard.vue`**: Improve `h2` so the title is readable (reduce text-transform to none or title-case). Increase `p` line height to 1.6. Limit body preview to a comfortable 3-line clamp with `overflow: hidden` and `-webkit-line-clamp: 3`. Change the "Edit" link to a secondary style (outline, not filled) so the card is read-first and edit is secondary.
- **`PostList.vue`**: Add padding inside each `<li>` so cards have breathing room. Remove the hard bottom border on individual list items and instead rely on PostCard's own margin.

## Acceptance Criteria

- `Layout.vue` body element uses a readable font family (e.g. `Georgia, "Times New Roman", serif` or `system-ui, sans-serif`) distinct from the display font used in `.site-header`
- `Layout.vue` `<head>` includes a `<title>` tag
- `.site-header` in `Layout.vue` contains an `<a>` element linking to `/`
- `PostCard.vue` body text `<p>` has `line-height: 1.6` or greater
- `PostCard.vue` body text is clamped to 3 visible lines via CSS line-clamp (or equivalent `max-height` + `overflow: hidden`)
- `PostCard.vue` "Edit" link is visually secondary (e.g. outline style or lower-contrast) compared to the post title
- All existing server tests (`go test ./examples/blog/...`) continue to pass after the changes
