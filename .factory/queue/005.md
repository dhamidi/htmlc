# Convert Renderer to write to io.Writer

## Description

The Renderer currently builds HTML into a `strings.Builder` and returns the result as a string from `Render()`. This spikes memory for large pages because the entire output must be held in a single allocation before the caller can use it. Convert the Renderer's public and internal APIs to write directly to an `io.Writer`, and add a `RenderString` convenience method for callers that still need a string.

### Changes to `renderer.go`

1. Change the `Render` method signature from `Render(scope map[string]any) (string, error)` to `Render(w io.Writer, scope map[string]any) error`. Move style collection and prop validation before writing. Write output directly to `w`.

2. Add a `RenderString(scope map[string]any) (string, error)` convenience method that creates a `strings.Builder`, calls `Render`, and returns the string.

3. Change all internal render methods (`renderNode`, `renderChildren`, `renderElement`, `renderRaw`, `renderVFor`, `renderConditionalChain`, `renderComponentElement`, `interpolate`) to accept `io.Writer` instead of `*strings.Builder`. Replace `sb.WriteString(s)` with `io.WriteString(w, s)` and `sb.WriteByte(b)` with `w.Write([]byte{b})`.

4. In `renderComponentElement`, slot pre-rendering still needs an internal `strings.Builder` (because the slot HTML is injected as the `$slot` scope variable). The child component's `Render` call should write to the parent writer directly — only the slot buffer is local.

5. Change the package-level convenience function `Render(c *Component, scope map[string]any) (string, error)` to `RenderString(c *Component, scope map[string]any) (string, error)`, and add `Render(w io.Writer, c *Component, scope map[string]any) error` that writes to the writer.

6. Update all tests in `renderer_test.go` to use the new signatures (`RenderString` for tests that compare output strings).

## Acceptance Criteria

- `Renderer.Render(w io.Writer, scope)` writes HTML directly to the provided `io.Writer` and returns only an error.
- `Renderer.RenderString(scope)` returns `(string, error)` and produces identical output to the old `Render`.
- Package-level `Render(w, c, scope)` writes to `io.Writer`; package-level `RenderString(c, scope)` returns a string.
- All internal render methods accept `io.Writer` instead of `*strings.Builder`.
- Slot pre-rendering in `renderComponentElement` still works correctly (uses a local buffer).
- All existing tests in `renderer_test.go` pass after being updated to the new API.
- `go vet ./...` and `go build ./...` pass with no errors.
