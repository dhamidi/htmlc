# Update Props() to handle v-slot binding patterns

## Description

Update the `Component.Props()` method in `component.go` to correctly handle `v-slot` binding patterns. Variables introduced by `v-slot` or `#` directives are locally scoped to the slot content (similar to `v-for` loop variables) and must be excluded from the component's prop list.

**Changes to `walkForProps` in `component.go`:**

The `walkForProps` function already tracks `v-for` variables as `childLocals` to exclude them from props. Apply the same pattern for `v-slot` bindings:

1. When encountering a `<template>` element with a `v-slot:name` or `#name` attribute, parse the binding pattern from the attribute value.
2. Add the bound variable names to a `childLocals` set for that template's children.
3. When encountering a component element (starts with uppercase) with `v-slot` or `#default` on the tag itself, parse the binding pattern and add bound variables to `childLocals` for the component's children.
4. Also recognize `#` shorthand in attribute scanning — currently `walkForProps` only looks for `v-` prefixed and `:` prefixed attributes. Add recognition of `#name` as a slot directive (not a prop source).

**What this means for prop discovery:**

Given this template:
```html
<UserList :users="users">
  <template #item="{ user, index }">
    <li>{{ index }}: {{ user.name }}</li>
  </template>
</UserList>
```

- `users` is a prop (referenced in `:users="users"` on the component tag)
- `user` and `index` are NOT props (they come from the slot binding pattern, locally scoped)

Given this template:
```html
<Wrapper v-slot="{ msg }">
  <p>{{ msg }}</p>
</Wrapper>
```

- `msg` is NOT a prop (comes from the slot binding on the component tag)

## Acceptance Criteria

- `Props()` excludes variables introduced by `v-slot` destructured bindings on `<template>` elements (e.g. `{ user, index }` from `<template #item="{ user, index }">`)
- `Props()` excludes variables introduced by `v-slot` single variable bindings on `<template>` elements (e.g. `props` from `<template #default="props">`)
- `Props()` excludes variables introduced by `v-slot` bindings on component tags (e.g. `{ msg }` from `<Wrapper v-slot="{ msg }">`)
- `Props()` correctly identifies props from `:attr="expr"` bindings on the component tag alongside `v-slot`
- `Props()` recognizes `#name` shorthand and does not treat the `#name="..."` attribute value as a prop expression
- New tests in `component_test.go`:
  - Template with `<template #item="{ user, index }">` using `{{ user.name }}` and `{{ index }}` → `user` and `index` are not in Props()
  - Template with `<Wrapper v-slot="{ msg }">{{ msg }}</Wrapper>` → `msg` is not in Props()
  - Template with `<UserList :users="users"><template #item="{ user }">{{ user.name }}</template></UserList>` → `users` IS in Props(), `user` is NOT
- All existing tests continue to pass
