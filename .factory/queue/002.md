# Add PropInfo type and Component.Props method

## Description

Add a `PropInfo` type and a `Props()` method on `*Component` in `component.go`. The method walks the component's parsed template AST and returns all top-level variable references (props) that the template uses, along with the expression strings where each prop appears.

### PropInfo type

```go
type PropInfo struct {
    Name        string
    Expressions []string
}
```

### Component.Props() method

The method walks the template tree and scans all expression sources:

| Source | How to extract the expression |
|---|---|
| `{{ expr }}` in text nodes | Match `\{\{(.*?)\}\}` on `TextNode.Data` |
| `:attr="expr"` | Attribute key starts with `:`, value is the expression |
| `v-bind:attr="expr"` | Attribute key starts with `v-bind:`, value is the expression |
| `v-if="expr"` | Attribute value |
| `v-else-if="expr"` | Attribute value |
| `v-show="expr"` | Attribute value |
| `v-text="expr"` | Attribute value |
| `v-html="expr"` | Attribute value |
| `v-for="vars in expr"` | Right side of ` in ` is the collection expression |

For each expression, call `expr.CollectIdentifiers` and add any identifier not in the exclusion set or the current local variable set to the result.

**Excluded identifiers:**
- Any identifier starting with `$` (e.g. `$slot`)
- `len` (built-in function)

**Scope tracking for v-for:**
- `v-for="item in items"` → `item` is local to the subtree.
- `v-for="(item, index) in items"` → `item` and `index` are local.
- `v-for="(value, key) in obj"` → `value` and `key` are local.
- `v-for="(value, key, index) in obj"` → `value`, `key`, and `index` are local.
- The collection expression is scanned in the **parent** scope (before adding v-for locals).
- All child nodes and non-v-for attributes of the v-for element are scanned with the v-for variables added to the local set.

Add tests in `component_test.go` covering:

- Simple interpolation: `{{ title }}` → prop `title`
- Bound attribute: `:class="cls"` → prop `cls`
- v-bind attribute: `v-bind:href="url"` → prop `url`
- Directive expressions: `v-if="show"`, `v-show="visible"`, `v-text="msg"`, `v-html="raw"`
- v-for scoping: `v-for="item in items"` → prop `items`, NOT `item`
- v-for with index: `v-for="(item, index) in list"` → prop `list`, NOT `item` or `index`
- Nested v-for: inner loop referencing outer loop variable → outer variable is not a prop
- `$slot` is excluded
- `len` is excluded
- Multiple expressions referencing the same prop aggregate in `Expressions`

## Acceptance Criteria

- `PropInfo` type is exported with `Name` and `Expressions` fields.
- `Component.Props()` returns the correct set of props for templates with interpolations, bindings, directives, and v-for scoping.
- v-for loop variables are correctly excluded from the prop set within their subtree.
- `$`-prefixed identifiers and `len` are excluded.
- All new tests in `component_test.go` pass.
