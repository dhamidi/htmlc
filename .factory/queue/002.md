# Update expr tests for removed `len` builtin

## Description

After removing `len` from the builtins map (task 001), two tests in `expr/eval_test.go` become stale:

- `TestBuiltinLen` — verifies `len(items)` returns the slice length; must be rewritten to verify that `len` is no longer callable and that `.length` is the correct alternative.
- `TestBuiltinLenStrictEqual` — verifies `len(posts) === 0`; this behavior is already covered by the existing `TestSliceLengthStrictEqual` test using `.length`; remove `TestBuiltinLenStrictEqual`.

Also add a new `TestRegisterBuiltin` test that verifies the exported `RegisterBuiltin` function works correctly: a custom function registered before evaluation is callable in expression strings.

Files to change: `expr/eval_test.go`

## Acceptance Criteria

- `TestBuiltinLen` is replaced by a test that verifies calling `len(items)` now returns an error (len resolves to undefined, which is not callable)
- `TestBuiltinLenStrictEqual` is deleted (covered by `TestSliceLengthStrictEqual`)
- A new `TestRegisterBuiltin` test registers a custom function and verifies it can be called from an expression
- `go test ./expr/...` passes with zero failures
