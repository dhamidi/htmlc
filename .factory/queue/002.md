# Implement collectSlotDefs — Scan Caller Children into Slot Buckets

## Description

Add `collectSlotDefs(parent *html.Node, scope map[string]any) map[string]*SlotDefinition` to `renderer.go`. This function is called by `renderComponentElement` to separate the component's direct children into named slot buckets before the child component's template is rendered.

**Algorithm:**

1. Iterate over `parent`'s direct child nodes.
2. For each child, check whether it is an element node whose tag is `<template>` **and** whose attributes include a recognised slot directive (`v-slot`, `v-slot:name`, `#name`).
   - If so, the child is a *named slot template*: extract the slot name via `parseSlotDirective`, parse the attribute value via `parseBindingPattern`, and create a `SlotDefinition` whose `Nodes` are the `<template>` element's own children (not the `<template>` element itself — it is a transparent wrapper), whose `ParentScope` is a shallow clone of `scope`, and whose `BindingVar`/`Bindings` come from the parsed pattern.
3. Any child node that does **not** match the above pattern is appended to the *default slot* bucket.
4. If any default-slot nodes were collected, produce a `SlotDefinition` for `"default"` with those nodes, the cloned parent scope, and no binding (binding is determined by the caller's component tag, handled separately in `renderComponentElement`).
5. Return the map; an empty map (not nil) is acceptable when there are no children.

**Edge cases to handle:**
- A `<template>` element with no slot directive is treated as a regular default-slot node (it is a transparent wrapper, but its slot membership is default).
- Whitespace-only text nodes between `<template>` elements are silently included in the default slot; they will render as whitespace, which is harmless.
- If two `<template>` elements target the same named slot, the *last* one wins (consistent with how duplicate HTML attributes behave elsewhere in the renderer).

## Acceptance Criteria

- `collectSlotDefs` is implemented in `renderer.go` and used by `renderComponentElement`.
- Children that are `<template v-slot:name>` elements produce a named `SlotDefinition` whose `Nodes` are those template's children (not the `<template>` node itself).
- Children that are `<template #name>` (shorthand) are treated identically to `<template v-slot:name>`.
- Children that are not slot templates accumulate into the `"default"` bucket.
- When the same slot name appears twice, the second definition replaces the first.
- An empty component element (no children) returns an empty map with no `"default"` entry.
- The returned `SlotDefinition.ParentScope` is a **clone** of the input scope (mutations during child rendering must not affect the caller's scope).
- Tests cover: single named slot, multiple named slots, default + named mix, duplicate named slot (last wins), no children.
