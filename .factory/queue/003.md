# Add missing prop validation to Renderer

## Description

Add missing-prop validation to the `Renderer` so that rendering fails (or invokes a handler) when a component's template references a prop not present in the render scope.

### New types and functions in `renderer.go`

**MissingPropFunc type:**

```go
// MissingPropFunc is called when a prop expected by the component's template
// is not present in the render scope. It receives the prop name and returns
// a substitute value, or an error to abort rendering.
type MissingPropFunc func(name string) (any, error)
```

**SubstituteMissingProp function:**

```go
// SubstituteMissingProp returns a placeholder string "MISSING PROP: <name>"
// for any missing prop.
func SubstituteMissingProp(name string) (any, error) {
    return fmt.Sprintf("MISSING PROP: %s", name), nil
}
```

**WithMissingPropHandler method on Renderer:**

```go
func (r *Renderer) WithMissingPropHandler(fn MissingPropFunc) *Renderer
```

Store the handler on the `Renderer` struct. Returns `*Renderer` for chaining (consistent with existing `WithStyles` and `WithComponents`).

### Validation logic in Render()

When `Renderer.Render(scope)` is called:

1. Call `r.component.Props()` to get the expected props.
2. For each `PropInfo`, check if `Name` exists as a key in `scope`.
3. If a prop is missing:
   - If `MissingPropHandler` is set: call it, inject the returned value into a copy of the scope.
   - If no handler is set: return an error `fmt.Errorf("missing prop %q (used in: %s)", name, strings.Join(expressions, ", "))`.
4. Proceed with rendering using the (possibly augmented) scope.

### Validation in renderComponentElement()

When rendering a child component via `renderComponentElement`, the child scope is built from attributes. Before rendering the child, validate the child's `Props()` against that scope using the same logic. The `MissingPropFunc` must be propagated to child renderers created in `renderComponentElement`.

### Tests in `renderer_test.go`

- Rendering with all props provided succeeds (no regression).
- Rendering with a missing prop and no handler returns an error mentioning the prop name and the expression(s) where it is used.
- Rendering with `SubstituteMissingProp` outputs `"MISSING PROP: <name>"` placeholders in the HTML.
- Rendering with a custom handler that returns `(nil, nil)` succeeds (treats missing props as nil).
- Rendering with a custom handler that returns an error propagates that error.
- Child component missing prop validation works (prop missing from attributes passed to child).

## Acceptance Criteria

- `MissingPropFunc` type, `SubstituteMissingProp`, and `WithMissingPropHandler` are exported.
- `Render()` validates scope against `Props()` and fails with a descriptive error when a prop is missing and no handler is set.
- When a handler is set, it is called for each missing prop and the returned value is injected into the scope.
- `SubstituteMissingProp` produces `"MISSING PROP: <name>"` placeholder strings.
- Missing prop validation also applies at child component boundaries in `renderComponentElement`.
- All new tests in `renderer_test.go` pass.
- Existing tests continue to pass (the `Render` free function and existing test helpers must not break).
