# Blog example: HTTP server

## Description

Create the HTTP server for the blog example in `examples/blog/server.go` (and a `examples/blog/main.go` entry point). The server wires together the `Store`, the `htmlc` engine, and `net/http` route handlers to deliver a working multi-page blog.

The server must handle these routes:

| Method | Path | Description |
|--------|------|-------------|
| GET | `/` | List all posts using `PostList.vue` inside `Layout.vue` |
| GET | `/posts/new` | Render the create-post form using `PostForm.vue` inside `Layout.vue` |
| POST | `/posts/new` | Parse form fields `title` and `body`, call `store.Create`, redirect to `/` |
| GET | `/posts/{id}/edit` | Render the edit-post form using `PostForm.vue` inside `Layout.vue`, pre-populated with existing post data |
| POST | `/posts/{id}/edit` | Parse form fields, call `store.Update`, redirect to `/` |

Route matching for `{id}` may use `net/http`'s built-in path value API (`r.PathValue("id")`) available in Go 1.22+, or simple string splitting — whichever is simpler given the module's Go version.

The server struct (`Server`) must:
- Hold a `*Store` and an `htmlc.Engine`
- Load component templates from `examples/blog/templates/` at startup using `htmlc.New(htmlc.Options{ComponentDir: ...})`
- Use `engine.RenderFragmentString` or `engine.RenderPageString` to render responses
- Write `text/html; charset=utf-8` responses
- Redirect (HTTP 303) after successful POST

The `main.go` entry point must start the server on `:8080` (or a port configurable via a `PORT` environment variable).

## Acceptance Criteria

- `examples/blog/server.go` defines a `Server` struct with `NewServer(store *Store, templateDir string) (*Server, error)` constructor
- `NewServer` initialises an `htmlc` engine from the given template directory
- `(*Server) Routes() http.Handler` returns a handler that serves all five routes listed above
- GET `/` renders a page listing all posts (or an empty-state message when there are none)
- GET `/posts/new` renders a blank create form
- POST `/posts/new` creates a post and redirects to `/`
- GET `/posts/{id}/edit` renders a pre-filled edit form; returns HTTP 404 for unknown IDs
- POST `/posts/{id}/edit` updates the post and redirects to `/`; returns HTTP 404 for unknown IDs
- `examples/blog/main.go` starts the HTTP server and logs the listening address
- `go build ./examples/blog/` succeeds
