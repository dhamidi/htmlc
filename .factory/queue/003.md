# Remove `len` from props discovery exclusion list in component.go

## Description

`component.go`'s `collectExprIdents` hard-codes `name == "len"` as a reason to skip an identifier when building the component's prop list. This exclusion exists solely because `len` was a built-in function and therefore should not be treated as an external dependency (prop). Now that `len` is no longer a builtin (task 001), the exclusion is wrong: if a template uses `len` as an identifier it must be detected as a required prop.

Remove the `name == "len"` condition from `collectExprIdents`. Update the companion test `TestProps_ExcludeLen` in `component_test.go` to reflect the new behaviour: the test should verify that `len` **is** now detected as a prop when referenced in a template expression (the old assertion that `len` must be absent should be inverted or replaced).

Files to change: `component.go`, `component_test.go`

## Acceptance Criteria

- `collectExprIdents` no longer contains the literal string `"len"` as an exclusion condition
- A component template containing `{{ len(items) }}` has both `len` and `items` in its `Props()` result
- `TestProps_ExcludeLen` is renamed or rewritten to assert that `len` appears in props (not that it is absent)
- `go test ./...` passes with zero failures
