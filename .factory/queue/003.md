# expr: write how-to documentation

## Description

Add how-to guide documentation for the `expr` package as `Example*` test functions in `expr/example_test.go`. The how-to quadrant of Diátaxis is goal-oriented: it shows how to accomplish specific, real tasks. It assumes the reader already knows the basics and wants to get something done.

Each `Example*` function should target a concrete use-case that a caller of the `expr` package (i.e., a developer extending `htmlc`) would actually encounter. The function body is the guide; the `// Output:` comment is the verifiable result.

How-to guides to include:

1. **`ExampleParse_memberAccess`** — evaluate a dot-chained expression against a nested Go map: `user.address.city` where `user` is a `map[string]any`.

2. **`ExampleParse_ternary`** — use a ternary to produce a CSS class string: `active ? "btn-active" : "btn"`.

3. **`ExampleParse_filterWithLen`** — use `len()` inside a conditional: `len(items) > 0 ? items[0] : null`.

4. **`ExampleParse_undefinedKey`** — show that accessing a missing key returns `UndefinedValue`, not `nil`, and that `UndefinedValue` is falsy.

5. **`ExampleParse_logicalShortCircuit`** — demonstrate `||` for default values: `title || "Untitled"`.

6. **`ExampleCollectIdentifiers`** — show how to extract the identifiers an expression depends on, which is useful for prop discovery.

Each example must:
- Import only `expr` (internal package path)
- Compile and pass `go test ./expr/` without modification
- Have a `// Output:` block that `go test` verifies automatically
- Include a one-sentence prose comment above the function explaining the goal (the "how-to" framing)

## Acceptance Criteria

- `expr/example_test.go` exists and contains all six `Example*` functions listed above
- `go test ./expr/` passes with all examples verified via `// Output:` blocks
- Each function has a goal-oriented one-sentence comment (not a description of what the code does, but what the reader is trying to achieve)
- No function duplicates an existing example already present in `expr/*_test.go`
