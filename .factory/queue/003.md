# Blog example: Vue component templates

## Description

Create the Vue component templates for the blog example application inside `examples/blog/templates/`. These components demonstrate named slots and scoped slots using the `htmlc` renderer.

The following `.vue` files are required:

**`Layout.vue`** — page shell with named slots:
- `<slot name="header">` — rendered inside a `<header>` element
- default `<slot>` — rendered inside a `<main>` element
- `<slot name="footer">` — rendered inside a `<footer>` element
- Provides fallback content for each slot (e.g. "Untitled" for header, copyright notice for footer)

**`PostList.vue`** — list of posts using a scoped slot:
- Accepts a `posts` prop (array of post objects)
- Renders an empty-state message via `v-if="posts.length === 0"`
- Iterates posts with `v-for` and exposes each post and its index to a `<slot name="item" :post="post" :index="index">` scoped slot
- Callers provide the item rendering template via `<template #item="{ post, index }">`

**`PostCard.vue`** — card for a single post:
- Accepts `id`, `title`, `body`, `createdAt` props
- Renders title, truncated body, and links to the edit page (`/posts/{{ id }}/edit`)
- Uses `v-text` or interpolation for all dynamic content

**`PostForm.vue`** — create/edit form:
- Accepts `post` prop (may be nil/absent for new post), `action` prop (form POST URL), and `submitLabel` prop
- Renders an HTML `<form>` with `method="POST"` and `:action="action"`
- Includes `<input name="title">` pre-filled with `{{ post.title }}` when editing
- Includes `<textarea name="body">` pre-filled with `{{ post.body }}` when editing
- Renders a submit button using `{{ submitLabel }}`

## Acceptance Criteria

- Files `Layout.vue`, `PostList.vue`, `PostCard.vue`, `PostForm.vue` exist under `examples/blog/templates/`
- `Layout.vue` uses `<slot name="header">`, default `<slot>`, and `<slot name="footer">` with fallback content in each
- `PostList.vue` uses `v-if` for empty state and `<slot name="item" :post="post" :index="index">` inside a `v-for` loop
- `PostCard.vue` renders `title`, `body`, and an edit link using the `id` prop
- `PostForm.vue` renders a `<form>` with `action`, `title` input, `body` textarea, and submit button
- All four files are valid `.vue` single-file component format (i.e., parseable by `htmlc.ParseFile`)
