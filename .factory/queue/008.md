# Document supported Vue.js subset

## Description

Update the project's `README.md` to provide a complete reference of exactly which Vue.js features htmlc supports, which it intentionally omits, and how the supported features differ from Vue.js behavior. The target audience is a Go developer with passing familiarity with Vue.js who wants to know what they can and cannot use.

The documentation should cover all of the following sections:

### 1. Overview

A brief explanation of what htmlc is: a server-side Go template engine that uses Vue.js Single File Component (`.vue`) syntax for authoring but renders entirely in Go with no JavaScript runtime. Emphasize that this is a static rendering engine — there is no reactivity, virtual DOM, or client-side hydration.

### 2. Template Syntax

#### Supported

- **Text interpolation**: `{{ expr }}` — evaluates the expression against the render scope and HTML-escapes the result. Multiple interpolations in one text node are supported.
- **Expression language**: Arithmetic (`+`, `-`, `*`, `/`, `%`, `**`), comparison (`===`, `!==`, `>`, `<`, `>=`, `<=`, `==`, `!=`), logical (`&&`, `||`, `!`), nullish coalescing (`??`), ternary (`? :`), member access (`obj.key`, `arr[i]`), function calls (`len(items)`), array literals (`[a, b]`), object literals (`{ key: value }`).
- **Built-in functions**: `len()` — returns the length of a string, array, slice, or map.

#### Not supported

- Filters (`{{ value | filterName }}`)
- JavaScript function definitions, arrow functions, `new`, `delete`
- Template literals (backtick strings)
- Optional chaining (`?.`), assignment operators, increment/decrement

### 3. Directives

For each directive, document the supported syntax and any behavioral differences from Vue.js.

| Directive | Supported | Notes |
|---|---|---|
| `v-text` | Yes | Sets element text content (HTML-escaped). Replaces children. |
| `v-html` | Yes | Sets element inner HTML (not escaped). Replaces children. |
| `v-show` | Yes | Adds `style="display:none"` when falsy. Merges with existing style. |
| `v-if` / `v-else-if` / `v-else` | Yes | Conditional rendering. Works on `<template>` for multi-element groups. |
| `v-for` | Yes | Supports arrays, maps, and integer ranges. Destructuring: `item in items`, `(item, index) in items`, `(value, key) in obj`, `(value, key, index) in obj`, `n in 5`. |
| `v-bind` / `:attr` | Yes | Dynamic attribute binding. `:class` supports object and array syntax. `:style` supports object syntax with camelCase-to-kebab conversion. Boolean attrs (disabled, checked, etc.) are omitted when falsy. |
| `v-pre` | Yes | Skips interpolation and directive processing for the element and all descendants. |
| `v-once` | Yes | No-op in server-side rendering (renders normally, directive is stripped). |
| `v-on` / `@event` | Passthrough | Preserved in output as-is for client-side JavaScript to handle. |
| `v-model` | Passthrough | Preserved in output as-is. |
| `v-slot` | No | Named slots and scoped slots are not supported. Only the default `<slot />` is available. |
| `v-cloak` | No | Not relevant for server-side rendering. |
| `v-memo` | No | |

### 4. Component System

#### Supported

- **Single File Components**: `.vue` files with `<template>`, optional `<script>` (preserved but not executed), optional `<style>`.
- **Props**: Passed via `:prop="expr"` (dynamic) or `prop="value"` (static string). No prop type validation or default values.
- **Default slot**: `<slot />` in a child component renders the caller's inner content. Slot content is evaluated in the caller's scope.
- **Component resolution**: Components are resolved by PascalCase name, kebab-case name, or case-insensitive match.
- **Nested composition**: Components can use other components from the same registry.
- **Scoped styles**: `<style scoped>` rewrites CSS selectors with a `data-v-*` scope attribute and adds the attribute to all HTML elements in the component.

#### Not supported

- Named slots / scoped slots / `v-slot`
- `<script setup>` or Composition API
- Computed properties, watchers, lifecycle hooks
- `$emit` / custom events
- `provide` / `inject`
- Dynamic components (`<component :is="...">`)
- Async components
- `defineProps` / `defineEmits` / `withDefaults`
- Teleport, Suspense, KeepAlive

### 5. Special Attributes

| Attribute | Behavior |
|---|---|
| `:key` | Rendered as `data-key="value"` in the output. |
| `class` + `:class` | Merged into a single `class` attribute. |
| `style` + `:style` | Merged into a single `style` attribute. |

### 6. Go API Quick Reference

Brief code snippets showing:
- `htmlc.New(Options{ComponentDir: "..."})` — create an engine
- `engine.RenderPage(name, data)` — render a full page with style injection
- `engine.RenderFragment(name, data)` — render an HTML fragment
- `engine.ServeComponent(name, dataFunc)` — create an `http.HandlerFunc`
- `htmlc.ParseFile(path, src)` — parse a `.vue` string
- `component.Props()` — discover expected props
- `engine.WithMissingPropHandler(fn)` — configure missing prop behavior
- `Options{Reload: true}` — enable hot-reload during development
