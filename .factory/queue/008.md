# Add Post Reading Detail Page to Blog

## Description

The blog currently has no route for reading a single post — only a list view and edit forms. Readers need a dedicated reading page that presents the full post title and body in a comfortable, focused layout. This task adds a `PostDetailPage.vue` template and a corresponding `GET /posts/{id}` route.

Changes:

- **`examples/blog/templates/PostDetailPage.vue`**: New page component. Uses `Layout` with `#header` slot containing the post title. Renders post body in a `<article>` element with reading-optimised CSS: `max-width: 680px`, `font-size: 1.125rem`, `line-height: 1.8`, and generous paragraph spacing. Includes a back link to `/` and an edit link to `/posts/{id}/edit`.
- **`examples/blog/templates/PostCard.vue`**: Add a link from the post title (the `<h2>`) to `/posts/{id}` so readers can navigate to the detail page from the list.
- **`examples/blog/server.go`**: Register `GET /posts/{id}` route. Render `PostDetailPage` with `post` prop set to the matching `Store.Get(id)` result. Return 404 when the post is not found.

## Acceptance Criteria

- `GET /posts/{id}` returns HTTP 200 and HTML containing the post title and body when the post exists
- `GET /posts/{id}` returns HTTP 404 when the post does not exist
- The rendered page includes a link back to `/` (home/list)
- The rendered page includes a link to `/posts/{id}/edit`
- `PostDetailPage.vue` uses `<article>` for the post body with `max-width` ≤ 680px and `line-height` ≥ 1.7
- `PostCard.vue` post title links to `/posts/{id}`
- All existing server tests (`go test ./examples/blog/...`) continue to pass; new route has at least one test covering the 200 and 404 cases
