# Add missing prop handler to Engine

## Description

Propagate the `MissingPropFunc` from the `Engine` to every `Renderer` it creates internally, so that missing-prop handling applies uniformly across all components rendered by the engine.

### New method on Engine

```go
// WithMissingPropHandler sets the function called when any component rendered
// by this engine has a missing prop. If not set, missing props cause render errors.
func (e *Engine) WithMissingPropHandler(fn MissingPropFunc) *Engine
```

Store the handler on the `Engine` struct. Return `*Engine` for chaining.

### Propagation

In `renderComponent` (the internal method that creates a `Renderer`), if the engine has a `MissingPropFunc` set, call `renderer.WithMissingPropHandler(fn)` before rendering.

### Tests in `engine_test.go`

- Engine without handler: rendering a component with a missing prop returns an error.
- Engine with `SubstituteMissingProp`: rendering a component with missing props produces placeholder output instead of an error.
- Engine with custom handler: the custom handler is invoked for missing props across all components (test with a parent component that includes a child component, both with missing props).
- Engine without handler: rendering a component with all props provided succeeds (no regression).

## Acceptance Criteria

- `Engine.WithMissingPropHandler` is exported and stores the handler on the engine.
- The engine propagates the handler to every `Renderer` it creates in `renderComponent`.
- All new tests in `engine_test.go` pass.
- Existing tests continue to pass.
