# Blog example: tests

## Description

Add tests for the blog example in `examples/blog/` that cover the store, the HTTP handlers, and the rendered HTML output end-to-end. Tests must use only the standard library (`testing`, `net/http/httptest`) and the `htmlc` module itself.

### Store tests (`examples/blog/store_test.go`)

- `TestStore_CreateAndList`: creates two posts and verifies `List` returns them in order with sequential IDs
- `TestStore_Get_Found`: creates a post and verifies `Get(id)` returns it with `ok=true`
- `TestStore_Get_NotFound`: verifies `Get` on an unused ID returns `(nil, false)`
- `TestStore_Update_Found`: creates a post, updates it, verifies new title/body are reflected
- `TestStore_Update_NotFound`: verifies `Update` on an unused ID returns `(nil, false)`

### HTTP handler tests (`examples/blog/server_test.go`)

Use `httptest.NewRecorder` and a temporary directory with the real `.vue` template files.

- `TestServer_ListEmpty`: GET `/` with no posts returns 200 and the empty-state message
- `TestServer_ListWithPosts`: GET `/` with two pre-created posts returns 200 and both post titles in the body
- `TestServer_NewForm`: GET `/posts/new` returns 200 and a `<form` element
- `TestServer_CreatePost`: POST `/posts/new` with valid form data returns 303 redirect to `/`; a subsequent GET `/` shows the new post title
- `TestServer_EditForm_Found`: GET `/posts/1/edit` after creating a post returns 200 and the post's title pre-filled in the form
- `TestServer_EditForm_NotFound`: GET `/posts/999/edit` returns 404
- `TestServer_UpdatePost`: POST `/posts/1/edit` with updated data returns 303; a subsequent GET `/` shows the updated title
- `TestServer_UpdatePost_NotFound`: POST `/posts/999/edit` returns 404

## Acceptance Criteria

- `examples/blog/store_test.go` contains all five store tests listed above and all pass
- `examples/blog/server_test.go` contains all eight HTTP handler tests listed above and all pass
- `go test ./examples/blog/` exits 0 with no failures
- Tests do not start a real network listener (use `httptest.NewRecorder` throughout)
